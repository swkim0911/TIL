# Reliable Byte Stream(TCP)
UDP같은 단순한 demultiplexing 프로토콜과 달리, **TCP는 신뢰성 있고, 연결 지향적이며, 바이트 스트림 형태의 서비스를 제공한다.** TCP는 누락되거나 순서가 뒤바뀐 데이터를 애플리케이션이 직접 처리할 필요가 없게 해주기 때문에 다양한 애플리케이션에서 매우 유용하다.

TCP의 특징을 보면, TCP는 바이트 스트림에 대한 신뢰적이고(reliable) 순서가 보장된 전송(in-ordered delivery)을 제공한다. 또한, TCP는 전이중(full-duplex) 프로토콜이기 때문에, 하나의 TCP 연결은 양 방향으로 흐르는 두 개의 바이트 스트림을 지원한다. 각각의 스트림에 대해, 수신 측이 한 번에 처리할 수 있는 데이터 양을 송신 측이 넘지 않도록 흐름 제어(flow control) 기능을 제공한다. 마지막으로 UDP와 마찬가지로, TCP 역시 한 호스트에서 여러 애플리리케이션이 동시에 서로 다른 통신을 수행할 수 있게 해주는 디멀티플랙싱(demultiplexing) 기능을 제공한다.

> full duplex: 두 장치가 동시에 데이터를 보내고 받을 수 있는 통식 방식

이러한 기능 외에도 TCP는 **혼잡 제어(congestion control)** 메커니즘을 갖고 있다. 이 메커니즘의 목적은 수신 측의 처리 능력을 초과하는 것을 막기 위한 것이 아니라, 네트워크가 감당할 수 있는 범위 내에서 송신 속도를 조절하여 네트워크 혼잡을 방지하는 것이다.

- 흐름 제어(flow control): 송신자가 수신자의 처리 능력을 초과하지 않도록 전송량을 조절하는 것 -> end to end issue
- 혼잡 제어(congestion control): 송신자가 네트워크 전체가 감당할 수 있는 양 이상으로 데이터를 밀어 넣지 않도록 제한하는 것 -> how hosts and networks interact
